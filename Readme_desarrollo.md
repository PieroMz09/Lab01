# 🧑‍💻 README - Desarrollo

## 📌 Objetivo

Este documento guía a desarrolladores para modificar, escalar o mantener el proyecto.

---

## 🛠️ Entorno de desarrollo

### Requisitos

* Editor de código (VS Code recomendado)
* Navegador moderno
* Servidor local (para PHP)

---

## 📁 Organización del código

### Frontend

* HTML → estructura
* CSS → estilos
* JS → interacción

### Backend

* PHP → manejo de formularios

---

## 🔧 Flujo de desarrollo

1. Clonar o descargar el proyecto
2. Abrir en editor
3. Ejecutar servidor local
4. Modificar archivos según necesidad

---

## 🎯 Convenciones

* Usar nombres de clases CSS descriptivos
* Mantener estructura modular
* Evitar duplicación de código

---

## ✏️ Edición de estilos

Ubicación:

```
/css/
```

Recomendaciones:

* Usar variables CSS (si se amplía)
* Mantener consistencia de colores

---

## ⚡ JavaScript

Ubicación:

```
/js/
```

Buenas prácticas:

* Separar lógica por funcionalidades
* Evitar código inline
* Usar eventos correctamente

---

## 🧩 Escalabilidad

Para convertir este proyecto en uno más robusto:

### Backend

* Migrar a:

  * Node.js / Express
  * Laravel
  * Django

### Base de datos

* MySQL / PostgreSQL

### Funcionalidades nuevas

* Login/registro
* Carrito persistente
* Pagos online

---

## 🧪 Debugging

* Usar consola del navegador
* Revisar errores de red (Network tab)
* Validar PHP logs

---

## 🚀 Deploy

### Opciones:

* Hosting tradicional (cPanel)
* VPS
* Plataformas cloud

---

## 📌 Recomendaciones finales

* Separar frontend y backend
* Implementar control de versiones (Git)
* Documentar cambios importantes
* Optimizar imágenes y recursos

---
